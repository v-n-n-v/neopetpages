petpage troubles

the "save changes" button isn't appearing when i try to save my petpage!

this is a very common problem. the easiest fix is to zoom out by hitting ctrl + the minus sign (or command + the minus sign, if you're using a mac) until the button reappears, usually near the top of the page and almost always in the center. however, this does not always work. the solution that does always work is a little more elaborate, and i've laid out the steps here:


(i know this looks complicated but it will take u like 2.5 seconds once u get the hang of it)

Chrome and Firefox

  1. Right click anywhere on the page
  2. Click inspect or inspect element
  3. In the block of code that shows up, scroll all the way up to the top until you find the initial body tag, under which you should see a div element with the id #THIS_ONE (not pictured in the screenshot below). if there is no div with the id #THIS_ONE, just start deleting every div elements of the form div id=[X] or div class=[X], starting from the top. especially ones with an id like #body or #main
  4. Right click on div id=THIS_ONE and hit delete element or delete node (Don't worry, this will not delete the element from the actual page that you're going to publish! It just temporarily removes it from view)
  5. The save changes button should appear! HOWEVER, if you got your code before 6/10/2019, then you will need to go down the list, deleting the div id=[X] or div class=[X] elements one by one until the "save changes" button appears. If it never appears, or if you're having trouble with these steps, please neomail me!

Safari

  1. Open Safari Preferences
  2. Select the "Advanced" tab all the way to the right and make sure that the box for "Show Develop menu in menu bar", at the bottom of the window, is checked
  3. Go back to the petpage you're trying to save and right click anywhere on the page
  4. Click inspect element
  5. In the block of code that shows up, scroll all the way up to the top until you find the initial body tag.
  6. Right click on the div id=THIS_ONE element (not pictured in screenshot) and hit delete ≫ node
  7. The save changes button should appear! NOTE: If you picked up the code before 6/10/2019, you won't find any div id=THIS_ONE — what you need to do is start deleting all the div elements of the form div id=[X] or div class=[X] until the button appears. If it never appears, or if you're having trouble with these steps, please neomail me!

Neopets is saying my code is longer than 4,000 characters!

chances are this is because the stylesheet has a bunch of unnecessary spaces, line breaks, and indents in it — this often happens if you put the code in a textarea (on a petpage, for example) without changing the pointy brackets in (style) and (/style) to round brackets, which provokes deranged code mynci to run through the code and reformat everything.

if u neomail me i can unformat it for u, but if u wanna try doing it urself, my method is to open just the stylesheet in a word doc — i.e. everything between (style) and (/style) — and make the following replacements (using ctrl + f for "find and replace"):


getting rid of unnecessary spaces:
  • replace , (comma plus a space) with , (comma by itself)
  • replace : (colon plus a space) with : (colon by itself)
  • replace { (space plus opening bracket) with { (opening bracket by itself)


getting rid of unnecessary line breaks and indents:
  • replace (opening bracket + line break + indent) with { (opening bracket by itself)
  • replace (semicolon + line break + indent) with ; (semicolon by itself)
  • replace (closing bracket + line break) with } (closing bracket by itself)


last step:
  • replace ;} with }.
